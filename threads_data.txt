東京美食地圖工具 實作計畫
目標說明
開發一個工具（Web App），用於彙整 Threads 討論串「第一屆東京沒吃到會後悔大賽」中所推薦的東京餐廳，並附上推薦理由。使用者可透過畫面定位自己的目前位置，並計算出距離最近的幾家推薦餐廳，同時提供直接連結至 Google Maps 的功能。

面臨的問題（需要您的協助）
WARNING

Threads 網站使用了強烈的防爬蟲機制與動態渲染，我嘗試自動抓取時遇到了阻擋（Browser Agent 暫時無法使用，且一般的爬蟲工具無法解析完整的留言內容）。 因此，需要請您手動複製該討論串的文字或留言內容貼給我，我會幫您把文字轉換為結構化的餐廳資料庫（包含名稱、理由、Google Maps 座標）。

Proposed Changes (實作方案)
我們將使用純前端技術 (HTML, CSS, JavaScript) 建立一個單頁面應用程式 (SPA)，這也方便您之後直接部署到 GitHub Pages。 專案將建立於 C:\Users\TK_Tsai\.gemini\antigravity\scratch\tokyo_food_map\ 之下。

核心檔案與功能
[NEW] index.html
建立具備現代感（Modern, Glassmorphism, 支援深色模式）的網頁介面。
包含一個「📍 尋找離我最近的餐廳」按鈕，觸發瀏覽器的 Geolocation API 取得使用者經緯度。
顯示餐廳卡片列表（名稱、推薦理由、距離、Google Map 導航按鈕）。
[NEW] style.css
實作質感佳的排版，使用現代字體 (如 Inter 或 Noto Sans TC)。
支援響應式設計 (RWD)，在手機版上有優良的體驗（因為使用者通常是用手機找餐廳）。
加入微動畫 (Micro-animations) 提升使用者互動體驗。
[NEW] app.js
負責邏輯處理：計算使用者位置與各餐廳的座標距離（使用 Haversine 公式）。
依據距離自動排序餐廳列表。
動態產生餐廳卡片並渲染至畫面上。
[NEW] data.js
將整理好的餐廳資料（包含名稱、理由、經緯度）以 JSON 格式儲存於此檔案中。
初期會先放入幾筆範例資料，待您提供討論串內容後，我會負責大量轉換並填入。
Verification Plan (驗證計畫)
手動驗證
用瀏覽器開啟 index.html，檢查介面美觀與排版。
點擊「定位」按鈕，確認瀏覽器是否跳出取得位置權限要求，並正確顯示計算出的距離（預期在台灣測試時，距離東京的餐廳會顯示約兩千多公里）。
點擊每家餐廳的 Google Map 連結，確認能正確開啓地圖並導向該餐廳。
